🌐 System Genesis: Full Ecosystem Blueprint
Project: LME Intelligence & Autonomous Macro-Agent System
Date of Genesis: July 2026

1. Technical Foundations (The "Core")
Dashboard: Built on Streamlit. Connects to lewiskhaw/lewis-metals-data (GitHub).

Data Flow: * office_exporter.py: Local Python script that crawls/processes Bloomberg data, cleans it (Hardened Alignment Layer), and pushes lme_master_data.csv to GitHub.

Sync: Streamlit uses GITHUB_TOKEN to pull live data for the dashboard.

Dashboard Features:

Quantitative Desk: Real-time metrics with delta monitoring, technical signal ingestion, and interactive Plotly charting with adaptive Y-axis scaling.

Ledger Engine: A dual-table view providing both raw data access (with conditional red/black formatting for spreads) and a processed "Monthly Average Pricing Analysis" (2026) for quick executive review.

2. The Agentic Layer (OpenClaw & Multi-Agent Intelligence)
We developed the OpenClaw agent framework, which functions as the intelligence backbone:

The News Nodes: Specialized ingestion agents for Indonesia_Mining_News, Global_Trade_Review, and China_Steel_Market.

The Synthesis Engine: Agents take raw input from the Nodes, analyze macroeconomic headwinds, and generate Markdown reports.

The Intelligence Feedback Loop: Reports are saved to 03_Case_Studies/. The Streamlit dashboard's Tab 2 automatically polls this folder to render intelligence reports for the user, bridging the gap between raw data and actionable macro analysis.

3. The Second Brain (Obsidian Graph)
Your Obsidian Vault is the qualitative center of the ecosystem. As shown in the graph view, the system is highly interconnected:

03_Case_Studies Folder: Houses the structured Markdown briefings generated by the agent.

02_Daily_Notes: Contains Unified_Market_Intelligence, where your agent synthesizes daily observations.

99_System Folder: Stores the AI_System_Prompt and Financial_Case_Study templates. This ensures that every report generated by the agents is consistent in structure and depth.

Knowledge Graph: The graph view shows how individual market events (e.g., "Indonesia Mining News") are linked to broader "Macro Shift Analysis" nodes, demonstrating the system's ability to map causal relationships between data points.

4. Operational Best Practices
Resilience: The system is built to handle node failures (like the Indonesia Mining Node failure) by defaulting to fallback research sources.

Data Integrity: The "Hardened Alignment Layer" in the Python scripts ensures that despite erratic source data, the dashboard always presents clean, numeric-formatted pricing.

Autonomous Scaling: The dashboard uses a "Claustrophobic Bounding" algorithm for Plotly charts—automatically recalculating the Y-axis range based only on the selected date window, which prevents legacy data from flattening your current charts.
